# Справка: контекст в Cursor (по документации)

Краткая выжимка из [Cursor Docs](https://cursor.com/docs) — что считается контекстом и что лучше давать Agent/Composer.

---

## Когда НЕ разбивать (Composer справится сам)

Composer автоматически:
- **Semantic search** — находит код по смыслу ("authentication flow" → auth-файлы)
- **Grep** — точный поиск
- **Dynamic context discovery** — подтягивает контекст по мере работы
- **Read, Edit, Terminal** — выполняет сам

Не разбивай: быстрые задачи, чёткий scope, знакомые паттерны. Composer найдёт нужные файлы сам.

## Когда разбивать

- Архитектурно сложно (много систем)
- Неясные требования
- Очень длинный горизонт (>10–15 шагов)

---

## Что Cursor называет контекстом

Всё, что модель видит в сессии:

1. **System prompt + Rules** — инструкции в начале
2. **User messages** — твои промпты и ответы
3. **@-упоминания** — файлы, папки, код, доки, прошлые чаты
4. **История диалога** — накапливается, при переполнении суммаризируется

---

## @-символы (Cursor 2.0)

| Символ | Назначение |
|--------|------------|
| **@Files & Folders** | Файл или папка — путь + обзор содержимого. Большие файлы конденсируются. |
| **@Code** | Конкретный фрагмент кода (точечный контекст) |
| **@Docs** | Документация (встроенная или своя по URL) |
| **@Past Chats** | Прошлые диалоги — для продолжения работы |

Убрано в 2.0: @Definitions, @Web, @Link, @Recent Changes, @Linter Errors — Agent сам подтягивает контекст.

---

## Rules (статический контекст)

- **Project Rules** — `.cursor/rules/`, в git
- **AGENTS.md** — в корне или подпапках
- **User Rules** — глобальные настройки
- **Team Rules** — для команд (dashboard)

Правила включаются в начало контекста модели. Рекомендации:
- До 500 строк
- Указывать на файлы, не копировать код
- Команды (npm run X), паттерны, канонические примеры

---

## Что Agent получает автоматически

- История текущего чата
- Rules (применяемые по globs/description)
- Файлы, которые ты @-упомянул
- **Semantic search** — Agent сам ищет по смыслу (grep + embeddings)
- **Read files** — читает файлы по необходимости

---

## Что Agent НЕ получает автоматически

- Документация пакетов (нужен @Docs)
- Полная git-история
- Закрытые/неупомянутые файлы

---

## Best practices (из Cursor Blog)

1. **Не тегать каждый файл** — если знаешь точный файл, укажи @. Иначе Agent найдёт через search.
2. **План перед кодом** — Plan Mode (Shift+Tab) для сложных задач.
3. **Правила — по мере необходимости** — добавляй, когда Agent повторяет одну и ту же ошибку.
4. **Конкретные промпты** — "add tests for auth.ts" хуже, чем "Write a test case for auth.ts covering the logout edge case, using patterns in __tests__/".
